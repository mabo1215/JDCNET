\section*{稿件信息}

\textbf{论文题目：} JDCNet: Confidence-Gated Privileged-Modality Distillation for Cost-Preserving X-ray Inference

\textbf{总体建议：} \textbf{Major Revision / Borderline Reject}

\textbf{总体评价：}
本文研究训练阶段使用 CT 作为 privileged modality、推理阶段仅使用 X-ray 的跨模态知识蒸馏问题。文章提出 JDCNet，即基于 CT 教师置信度门控的辅助监督机制，并在 510 个 same-patient CT--X-ray paired BIMCV 样本上进行 patient-level 5-fold cross-validation。论文优点是问题定义较清楚，部署成本意识较强，并且尝试用固定统计门限限制过度宣称。然而，以 TCSVT 的标准来看，当前版本仍存在显著问题：TCSVT 适配性需要进一步加强，方法新颖性边界不够清楚，超参数选择与统计检验可能存在选择偏差，实验仍主要依赖单一公开小规模 paired cohort，外部验证不足，系统级部署实验也不充分。因此，当前版本不建议直接接收，需要大幅修改后再考虑。

\section*{一、与 TCSVT 投稿要求的匹配度}

\subsection*{1. 范围匹配性}

TCSVT 关注图像/视频技术中的 acquisition、processing、analysis、learning、understanding、compression、transmission、systems implementation 等方向。本文虽然属于医学图像分类和视觉系统部署方向，但当前稿件更像一篇 medical imaging / cross-modal distillation paper，而不是典型的 video technology / circuits-and-systems paper。

\textbf{主要问题：}
\begin{enumerate}[leftmargin=2em]
    \item 论文标题和摘要强调 X-ray inference，而不是 video technology、visual systems 或 deployment system。
    \item 系统层面贡献目前主要停留在参数量和 MACs 对比，缺少真实 latency、memory、throughput、energy、edge-device deployment 等 TCSVT 读者更关注的系统指标。
    \item 医学任务是二分类 COVID-19 vs non-COVID，视觉系统普适性尚未充分证明。
\end{enumerate}

\textbf{修改建议：}
\begin{enumerate}[leftmargin=2em]
    \item 在 Introduction 中明确说明本文属于 \textit{cost-preserving visual inference systems}，而不仅是医学分类。
    \item 增加真实部署实验，例如 CPU、GPU、edge GPU 或 embedded platform 上的 latency、memory footprint、throughput 和 energy consumption。
    \item 增加一张 ``TCSVT positioning table''，说明本文相对于 cross-modal fusion、same-modality KD、medical image classification、efficient visual inference 的区别。
\end{enumerate}

\subsection*{2. 篇幅与格式}

当前稿件为 14 页，已经达到 TCSVT Transactions Paper 的页数上限。主文、参考文献和 appendix 都非常紧凑，部分表格信息密度较高，可读性不足。

\textbf{修改建议：}
\begin{enumerate}[leftmargin=2em]
    \item 将 Table 8、Table 9 和部分 supplementary interpretation 移入 supplementary material。
    \item 主文保留关键结果表、部署成本表、主要 ablation 表即可。
    \item 压缩过长的 Related Work，避免将过多未直接影响实验设计的文献讨论放在正文。
    \item 提高 Figure 1 的清晰度，使用更符合 IEEE 风格的矢量图，并减少图内过小文字。
\end{enumerate}

\section*{二、论文贡献总结}

本文的核心贡献可以概括为：

\begin{enumerate}[leftmargin=2em]
    \item 提出训练阶段使用 CT、推理阶段仅使用 X-ray 的 cost-preserving privileged-modality learning 问题设定。
    \item 提出 JDCNet，即使用 CT teacher confidence gate 选择可靠样本，并对 X-ray student 施加 hard target 或 soft-KL auxiliary target。
    \item 在 510 个 same-patient CT--X-ray BIMCV paired cases 上进行 patient-level 5-fold cross-validation，并报告两个 JDCNet 配置通过固定 transfer gate。
    \item 比较 gated logit KD、contrastive alignment、attention transfer、feature hints、BiomedCLIP 等替代机制，认为 confidence-gated auxiliary target 是目前唯一通过统计门限的机制。
\end{enumerate}

这些贡献具有一定研究价值，但目前更像是一个谨慎的机制审计研究，而不是一个充分成熟的 TCSVT Transactions Paper。若要达到 TCSVT 水平，需要更强的新颖性论证、更严格的统计验证、更充分的系统实验和外部泛化验证。

\section*{三、主要优点}

\begin{enumerate}[leftmargin=2em]
    \item \textbf{问题设定有现实意义。} 使用 CT 作为 training-only privileged modality，同时保持 X-ray-only deployment，是一个有实际部署价值的问题。
    
    \item \textbf{部署成本意识较强。} 论文明确区分训练阶段教师分支和推理阶段学生分支，并强调 JDCNet student 与 supervised ResNet-18 baseline 具有相同参数量和 MACs。
    
    \item \textbf{评价协议比一般小样本医学图像论文更谨慎。} 使用 patient-level splits、balanced accuracy、paired difference、bootstrap confidence interval 和 fold/seed win counts，比单次随机划分更可靠。
    
    \item \textbf{负结果报告较完整。} 文章不仅报告 JDCNet 的正结果，也报告 logit KD、contrastive alignment、BiomedCLIP 等机制未通过 gate，这有助于界定贡献边界。
    
    \item \textbf{限制陈述较诚实。} 论文多次强调结果仅限于单一 public paired BIMCV cohort，不宣称 clinical readiness，这一点是积极的。
\end{enumerate}

\section*{四、主要问题与必须修改项}

\subsection*{Major Concern 1: TCSVT 适配性仍然偏弱}

\textbf{问题：}
当前论文的核心任务是医学影像二分类，且数据规模较小。虽然作者强调 cost-preserving X-ray inference，但对 TCSVT 而言，仅报告参数量和 MACs 不足以证明这是一个系统级 visual technology contribution。文章需要更明确地回答：为什么这个工作应投 TCSVT，而不是 TMI、JBHI、Medical Image Analysis 或 IEEE Access？

\textbf{必须修改：}
\begin{enumerate}[leftmargin=2em]
    \item 在 Introduction 末尾增加一个 TCSVT-specific positioning paragraph。
    \item 在 Experiments 中增加真实系统指标，包括：
    \begin{enumerate}[leftmargin=2em]
        \item CPU latency；
        \item GPU latency；
        \item batch size = 1 的 real-time inference latency；
        \item peak memory；
        \item preprocessing cost；
        \item training-time overhead；
        \item teacher branch removal 后的实际 deployment graph。
    \end{enumerate}
    \item 增加与 lightweight backbones 的比较，例如 MobileNetV2、EfficientNet-B0、ConvNeXt-Tiny 或 ViT-Tiny，以证明该方法不仅适用于 ResNet-18。
\end{enumerate}

\textbf{建议加入的英文表述：}
\begin{quote}
To better align with the scope of TCSVT, we position the proposed method as a cost-preserving visual inference system rather than a clinical diagnostic model. The CT branch is used only to shape the training objective, while the deployed graph remains a standard single-modality X-ray classifier. Therefore, the central systems question is whether privileged-modality supervision can improve the deployed visual model without increasing inference-time modality acquisition, memory footprint, or latency.
\end{quote}

\subsection*{Major Concern 2: 方法新颖性边界不够清楚}

\textbf{问题：}
JDCNet 的核心机制是 teacher confidence gate + auxiliary hard/soft target。这与 reliability-aware distillation、selective distillation、sample-wise KD、learning using privileged information、modality hallucination 等已有方向高度相关。当前稿件虽然引用了一些相关工作，但还没有充分说明 JDCNet 到底新在哪里。

\textbf{必须修改：}
\begin{enumerate}[leftmargin=2em]
    \item 增加一张方法对比表，至少包括以下维度：
    \begin{enumerate}[leftmargin=2em]
        \item 是否 cross-modal；
        \item 是否 training-only privileged modality；
        \item 是否 inference-time cost-preserving；
        \item 是否使用 teacher confidence；
        \item 是否使用 hard mask；
        \item 是否在 same-patient paired medical cohort 上验证；
        \item 是否报告 failed transfer mechanisms。
    \end{enumerate}
    \item 明确区分 JDCNet 与 gated logit KD 的数学差异和实验差异。
    \item 避免将 JDCNet 描述成全新网络架构。更准确的说法应是 ``confidence-gated privileged-modality distillation objective'' 或 ``training protocol''。
\end{enumerate}

\textbf{建议修改标题：}
\begin{quote}
Confidence-Gated Privileged-Modality Distillation for Cost-Preserving X-ray Inference
\end{quote}

该标题可以弱化 ``Net'' 带来的架构创新暗示，更符合本文实际贡献。

\subsection*{Major Concern 3: 超参数选择与统计检验存在潜在选择偏差}

\textbf{问题：}
论文报告 16 个 JDCNet 配置，其中 2 个配置通过固定 gate。虽然 gate 是固定的，但如果 $\tau_{\mathrm{gate}}$、$\lambda$、teacher view 和 target type 是在同一 5-fold CV 结果上选择出来的，那么 reported confidence interval 可能低估选择偏差。换句话说，``2/16 cells pass'' 不能直接等价于一个预注册假设通过验证。

\textbf{必须修改：}
\begin{enumerate}[leftmargin=2em]
    \item 明确说明 hyperparameter selection protocol：
    \begin{enumerate}[leftmargin=2em]
        \item $\tau_{\mathrm{gate}}$ 是否预先固定？
        \item $\lambda$ 是否预先固定？
        \item mid-slice 和 3-slice teacher 是否预先指定？
        \item hard target 和 soft-KL 是否预先指定？
        \item 是否使用 validation folds 调参？
    \end{enumerate}
    \item 若当前结果来自同一 CV 网格搜索，应补充 nested cross-validation 或 independent validation split。
    \item 对 16 个配置和多个 comparator 进行 multiple-comparison correction，或将结果改写为 exploratory evidence。
    \item 报告所有配置的 selection-adjusted confidence intervals 或使用 paired permutation test。
\end{enumerate}

\textbf{建议加入的英文表述：}
\begin{quote}
Because the JDCNet grid contains multiple teacher views, gate thresholds, auxiliary weights, and target types, the two passing configurations should be interpreted as selected configurations unless the hyperparameters were pre-specified before evaluation. We therefore report the full grid and, in the revised version, add a nested-selection analysis to estimate the transfer effect after accounting for hyperparameter selection.
\end{quote}

\subsection*{Major Concern 4: Bootstrap 单位可能不独立}

\textbf{问题：}
论文使用 $5$ folds $\times$ $3$ seeds 得到 $15$ paired fold--seed cells，并对这些 cells 做 bootstrap。该做法存在潜在问题：不同 seeds 和 folds 并不一定独立，尤其同一 patient 会在不同 seed 的不同 split 中重复出现。直接对 15 个 fold--seed cells bootstrap 可能会高估统计显著性。

\textbf{必须修改：}
\begin{enumerate}[leftmargin=2em]
    \item 增加 patient-level bootstrap 或 hierarchical bootstrap。
    \item 报告 per-patient prediction aggregation 后的 paired statistical test。
    \item 对 BA、AUC、macro-F1 分别报告 paired confidence intervals。
    \item 对 ROC-AUC 使用 DeLong test 或 paired bootstrap。
    \item 增加 per-fold/per-seed raw results table，避免均值掩盖不稳定性。
\end{enumerate}

\subsection*{Major Concern 5: 单一 public paired cohort 不足以支持较强泛化结论}

\textbf{问题：}
主结果完全依赖 510 个 BIMCV same-patient paired cases，其中正样本 113、负样本 397。虽然论文已经承认这一限制，但 TCSVT Transactions Paper 通常需要更强的泛化证据。尤其是 COVID-19 任务容易受到 source bias、scanner protocol、hospital-specific artifacts 和 label leakage 的影响。

\textbf{必须修改：}
\begin{enumerate}[leftmargin=2em]
    \item 增加至少一个 external paired cohort，哪怕规模较小，也应作为独立验证。
    \item 如果无法获得 external CT--X-ray paired cohort，应增加 cross-source stress test，并将主张明确降级为 within-cohort mechanism audit。
    \item 报告 CT 与 X-ray 的 acquisition time gap、pairing criteria、patient ID matching 逻辑。
    \item 说明是否存在同一患者多张 X-ray 或多组 CT series，如何避免 patient leakage。
    \item 报告 dataset construction code 或 manifest。
\end{enumerate}

\subsection*{Major Concern 6: 缺少 teacher confidence gate 的机制分析}

\textbf{问题：}
JDCNet 的核心是 confidence gate，但当前稿件没有充分报告 gate 本身的行为。读者无法判断该方法到底在选择哪些样本、是否偏向多数类、是否只是过滤困难样本、是否提高 calibration，或者是否引入 selection bias。

\textbf{必须补充：}
\begin{enumerate}[leftmargin=2em]
    \item 每个 $\tau_{\mathrm{gate}}$ 下的 gate coverage。
    \item positive class 和 negative class 的 gate coverage。
    \item 被 gate 选中样本与未选中样本的 teacher accuracy。
    \item teacher confidence calibration，例如 ECE、Brier score、reliability diagram。
    \item gate threshold sensitivity curve。
    \item JDCNet gain 与 gate coverage 的关系图。
\end{enumerate}

\textbf{建议新增表格：}
\begin{quote}
Table X: Gate statistics under different $\tau_{\mathrm{gate}}$: selected ratio, selected positive ratio, selected negative ratio, teacher accuracy on selected samples, teacher accuracy on rejected samples, and resulting $\Delta$BA.
\end{quote}

\subsection*{Major Concern 7: Comparator fairness 需要进一步证明}

\textbf{问题：}
论文声称 gated logit KD、contrastive alignment、attention transfer、feature hints、BiomedCLIP fine-tuning 等方法都未通过 gate。但这些 comparator 是否经过同等强度调参尚不清楚。如果 JDCNet 有 16-cell sweep，而 comparator 只有少量配置，则 comparison fairness 会受到质疑。

\textbf{必须修改：}
\begin{enumerate}[leftmargin=2em]
    \item 给出每个 comparator 的完整搜索空间。
    \item 报告 comparator 的最佳验证设置和调参预算。
    \item 确保所有 comparator 使用相同 backbone、相同 input resolution、相同 training epochs、相同 optimizer、相同 data augmentation。
    \item 对 BiomedCLIP 说明 fine-tuning protocol、frozen/unfrozen layers、learning rate、input normalization 和 prompt setting。
    \item 增加 same-modality X-ray teacher-to-X-ray student KD 作为 sanity check。
\end{enumerate}

\subsection*{Major Concern 8: 只报告 $\Delta$BA 不够，需要绝对性能和临床相关指标}

\textbf{问题：}
论文主要强调 JDCNet 相对 supervised X-ray baseline 的 $\Delta$BA，但缺少完整的 absolute performance。对于医学影像二分类，仅报告 BA 差值不足以判断模型是否有实际价值。

\textbf{必须补充：}
\begin{enumerate}[leftmargin=2em]
    \item Supervised X-ray、CT teacher、JDCNet、comparators 的 absolute BA、sensitivity、specificity、macro-F1、ROC-AUC、PR-AUC。
    \item 每个 fold 的 confusion matrix。
    \item ROC curve 和 precision-recall curve。
    \item 95\% CI 的 absolute metric table。
    \item PPV 和 NPV，尤其考虑 class imbalance。
\end{enumerate}

\subsection*{Major Concern 9: 部署成本实验不充分}

\textbf{问题：}
Table 6 报告了 params 和 MACs，但没有足够的真实推理成本。TCSVT 读者通常会关注真实系统性能，而不是仅关注 theoretical MACs。

\textbf{必须补充：}
\begin{enumerate}[leftmargin=2em]
    \item 在 RTX 3090、CPU 和一个 edge/embedded device 上报告 latency。
    \item 报告 batch size = 1 和 batch size = 16/32 的吞吐。
    \item 报告 preprocessing time。
    \item 报告 peak memory。
    \item 报告 teacher training overhead 和 student training overhead。
    \item 明确 JDCNet 在 inference graph 中没有 CT branch、没有 confidence gate、没有 auxiliary loss。
\end{enumerate}

\section*{七、建议重写的贡献表述}

当前贡献表述中 ``JDCNet'' 容易被理解为一个新网络结构，但实际贡献更接近训练目标和证据审计。建议改写如下：

\begin{quote}
\textbf{Contribution 1.} We formulate CT-to-X-ray learning as a training-only privileged-modality visual inference problem, where CT is available only during training and the deployed model remains an X-ray-only classifier.

\textbf{Contribution 2.} We propose a confidence-gated privileged-modality distillation objective that applies CT-derived auxiliary targets only to samples on which the teacher is sufficiently confident, while preserving the supervised X-ray loss on all samples.

\textbf{Contribution 3.} We evaluate the method under patient-level paired cross-validation and a pre-specified transfer gate, and we report both successful and failed transfer mechanisms to avoid over-interpreting small-cohort gains.

\textbf{Contribution 4.} We provide a deployment-cost analysis showing that the selected student has the same inference graph as the supervised X-ray baseline, and we identify the remaining need for external paired-cohort validation.
\end{quote}

\section*{八、建议新增的 Method Algorithm}

建议在 Method 中加入如下伪代码：

\begin{quote}
\textbf{Algorithm 1: Confidence-Gated Privileged-Modality Distillation}

\begin{enumerate}[leftmargin=2em]
    \item Train or load a CT teacher $f_T$.
    \item For each paired sample $(x_i^{xr}, x_i^{ct}, y_i)$:
    \begin{enumerate}[leftmargin=2em]
        \item Compute teacher probability $p_i^{ct} = \mathrm{softmax}(f_T(x_i^{ct}))$.
        \item Compute teacher confidence $c_i = \max_k p_{i,k}^{ct}$.
        \item If $c_i > \tau_{\mathrm{gate}}$, include sample $i$ in the auxiliary set $M$.
    \end{enumerate}
    \item Train the X-ray student $f_S$ using:
    \[
    \mathcal{L}
    =
    \mathcal{L}_{CE}(f_S(x^{xr}), y)
    +
    \lambda \cdot \mathbb{I}(i \in M)
    \mathcal{L}_{aux}(f_S(x^{xr}), f_T(x^{ct})).
    \]
    \item Discard the CT teacher and auxiliary branch at inference time.
\end{enumerate}
\end{quote}

\section*{九、最终审稿结论}

\textbf{结论：}
本文提出的问题有价值，论文整体写作也比较谨慎，尤其是对小样本 paired medical cohort 的证据边界有较清楚认识。然而，若按照 TCSVT Transactions Paper 的标准，当前稿件仍然不足以直接接收。最关键的问题包括：TCSVT 系统定位不够强，真实部署实验不足，JDCNet 的新颖性与已有 selective/reliability-aware distillation 的关系没有完全厘清，超参数选择和统计检验可能存在偏差，且结果仍缺乏外部 paired cohort 验证。

\textbf{推荐处理意见：}
\[
\boxed{\text{Major Revision}}
\]

\textbf{若作者能够补充以下内容，稿件质量会显著提高：}
\begin{enumerate}[leftmargin=2em]
    \item 增加真实 latency/memory/throughput 系统实验；
    \item 增加 nested validation 或选择偏差校正；
    \item 增加 gate coverage、teacher calibration 和 threshold sensitivity 分析；
    \item 增加 absolute metric table 和 per-fold raw results；
    \item 证明 comparator 调参公平；
    \item 尽可能补充 external paired cohort 或将 claim 明确降级为 within-cohort mechanism audit；
    \item 重新定位论文，使其更符合 TCSVT 的 visual systems 和 cost-preserving inference 范围。
\end{enumerate}
