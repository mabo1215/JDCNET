\section*{Section-by-Section Revision Checklist (Top-Tier Journal/Conference Standard)}

\subsection*{Overall Assessment}
\begin{itemize}
    \item The revised manuscript has already improved substantially in \textbf{honesty, reproducibility, and claim control}. In particular, the paper now explicitly states that:
    \begin{itemize}
        \item the study is a \textbf{reproducible pilot study},
        \item the paired validation split is \textbf{extremely small} (4 X-ray images),
        \item the \textbf{student-only baseline} is currently the strongest paired-cohort result,
        \item and the current \textbf{MHRA implementation is not yet robust}.
    \end{itemize}
    \item These changes are important and move the manuscript away from overclaiming.
    \item However, under top-tier standards, the manuscript still needs another round of refinement to make the contribution framing, methodological necessity, experimental rigor, and limitation-to-future-work logic more convincing.
\end{itemize}

\subsection*{1. Abstract}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The abstract now clearly states the task, the proposed framework, the dataset setting, and the key negative finding that cross-modality distillation does not yet stably outperform the student-only baseline.
        \item The wording is much more conservative and scientifically credible than a typical overclaimed medical-AI abstract.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item The current abstract lists DPE, MHRA, and DFPN, but it still does not make fully clear \emph{why these three modules are necessary together} rather than being a generic modular stack.
        \item The abstract should more sharply distinguish between:
        \begin{itemize}
            \item the \textbf{scientific question} (Can CT supervise X-ray through cross-modality distillation?),
            \item the \textbf{engineering artifact} (the executable JDCNet scaffold),
            \item and the \textbf{empirical conclusion} (no stable gain yet under the current small paired cohort).
        \end{itemize}
        \item For a top-tier venue, the last sentence should emphasize the paper's value as a \textbf{negative-but-reproducible benchmarked finding}, not just as an underperforming method.
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item Rewrite the final two sentences so that the contribution is framed as:
        \begin{quote}
            a reproducible cross-modality benchmark/pilot pipeline that prevents overstated claims and establishes a controlled baseline for future CT-to-X-ray distillation research.
        \end{quote}
    \end{itemize}
\end{itemize}

\subsection*{2. Introduction}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The introduction now has a clear motivation: CT provides richer supervision, while X-ray is the target deployment modality.
        \item The problem setting is more realistic than multi-modal fusion at test time.
        \item The contribution list is cleaner and now includes the executable evaluation pipeline.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item The introduction currently explains \emph{what} the paper does, but not strongly enough \emph{why this problem matters now} in the context of modern medical multi-modal learning.
        \item It still lacks a sharper statement of the \textbf{gap in prior work}: existing KD papers are mostly same-modality, while medical multi-modal papers often assume inference-time access to all modalities.
        \item The current contribution list is partially \textbf{method-oriented} and partially \textbf{pipeline-oriented}; this should be unified.
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item Add one paragraph at the end of the introduction explicitly stating:
        \begin{enumerate}
            \item what prior work has \emph{not} done,
            \item why cross-modality distillation is technically nontrivial,
            \item and why a reproducible negative/neutral result is still valuable.
        \end{enumerate}
        \item Rewrite the contributions into three layers:
        \begin{enumerate}
            \item \textbf{Problem formulation},
            \item \textbf{Executable framework},
            \item \textbf{Empirical finding and cautionary conclusion}.
        \end{enumerate}
    \end{itemize}
\end{itemize}

\subsection*{3. Related Work}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The related work is now logically divided into chest imaging, transformers/attention, knowledge distillation, and cross-modality/multi-modal learning.
        \item The manuscript correctly positions the paper closer to cross-modality distillation than to conventional multi-modal fusion.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item The section is still too \textbf{descriptive} and not sufficiently \textbf{comparative}.
        \item It does not yet clearly identify the exact methodological niche of JDCNet relative to:
        \begin{itemize}
            \item same-modality distillation,
            \item feature distillation,
            \item logit distillation,
            \item multi-modal fusion,
            \item and cross-modal representation transfer.
        \end{itemize}
        \item There is still a risk that reviewers will ask: ``Is this really a new method, or mainly a new experimental packaging of known blocks?''
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item End the related work with a \textbf{gap-summary paragraph} in this style:
        \begin{quote}
            Existing medical KD studies mostly focus on same-modality compression, whereas multi-modal medical studies often require all modalities at inference. In contrast, this work studies training-only CT supervision for X-ray deployment and evaluates that setting under an executable, patient-level, repeated-run protocol.
        \end{quote}
        \item If possible, add 1--3 stronger and more recent references specifically on \textbf{cross-modal distillation} or \textbf{teacher-student transfer across modalities}, even if they are outside chest imaging.
    \end{itemize}
\end{itemize}

\subsection*{4. Method}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The method section is structurally clear.
        \item The paper now gives equations for DPE, MHRA, DFPN, and the hard/soft distillation objective.
        \item The text explicitly states which parts are conceptual and which parts correspond to the current executable implementation.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item The current method is understandable, but the \textbf{methodological novelty is still not fully defended}.
        \item DPE, MHRA, and DFPN are introduced as intuitive modules, but their \textbf{necessity, interaction, and modality-specific role separation} are not yet argued strongly enough.
        \item Since the ablation shows that \textbf{w/o MHRA performs slightly better}, reviewers may question whether MHRA is conceptually justified or currently unnecessary.
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item Add a short paragraph called something like \textbf{``Design Rationale''} after the architecture overview.
        \item Explicitly explain:
        \begin{itemize}
            \item why teacher-side saliency refinement is needed before transfer,
            \item why student-side multi-scale fusion is especially relevant for X-ray,
            \item and why these modules were chosen instead of plain logit KD alone.
        \end{itemize}
        \item Add one sentence openly acknowledging:
        \begin{quote}
            although the present MHRA instantiation is not yet empirically validated, it is included as a testable hypothesis about teacher-side saliency aggregation rather than as a confirmed improvement.
        \end{quote}
        \item This kind of wording can preempt novelty and ablation criticisms.
    \end{itemize}
\end{itemize}

\subsection*{5. Experiments}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The experimental section is now much more reproducible.
        \item The manuscript clearly states cohort construction, patient-level splitting, repeated seeds, baseline types, optimizer settings, and executable pipeline generation.
        \item The paper also explicitly warns that the dataset should not be treated as a competition benchmark or clinical validation benchmark.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item Under top-tier standards, the experimental evidence is still the manuscript's weakest part.
        \item The main issue is not presentation quality, but \textbf{evidence strength}: the paired validation set has only 4 images, and the CT validation split is effectively unusable for meaningful comparison.
        \item The current baselines are reasonable, but the study still lacks stronger external anchors, such as:
        \begin{itemize}
            \item a plain student trained on all available X-ray data and evaluated under a matched protocol,
            \item a simpler cross-modal KD baseline without DPE/MHRA/DFPN,
            \item or a non-distillation transfer baseline.
        \end{itemize}
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item Strengthen the experiment section by explicitly separating:
        \begin{enumerate}
            \item \textbf{benchmarking experiments},
            \item \textbf{stress-test experiments},
            \item \textbf{reproducibility utilities}.
        \end{enumerate}
        \item Add one paragraph clarifying that the current experiment is designed to test \textbf{feasibility and reproducibility}, not to establish state-of-the-art performance.
        \item If no new data can be added, then make that limitation part of the experimental design logic rather than leaving it as a passive weakness.
    \end{itemize}
\end{itemize}

\subsection*{6. Results}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The results section is now much more honest and publishable than before.
        \item The paper correctly avoids overemphasizing unstable metrics and focuses on accuracy, macro-F1, and balanced accuracy.
        \item The discussion of the student-only baseline, the flat temperature/alpha ablation, and the unstable MHRA behavior is scientifically appropriate.
        \item The appendix usefully supports the main text with per-seed tables, qualitative error analysis, and seed-aggregated confusion summaries.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item The results are credible, but the \textbf{narrative structure} can still be stronger.
        \item Right now the section mainly says what happened; it should more explicitly tell the reader \textbf{what we learn from the failure to improve}.
        \item Top-tier reviewers often accept negative results only when the paper extracts a strong, transferable lesson.
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item Reorganize the results discussion around three takeaways:
        \begin{enumerate}
            \item \textbf{No stable gain from current cross-modality KD under the tiny paired cohort;}
            \item \textbf{the dominant failure mode is prevalence-biased positive prediction;}
            \item \textbf{module complexity is not justified unless the paired cohort and implementation quality are both improved.}
        \end{enumerate}
        \item This will make the paper read less like an unsuccessful method paper and more like a carefully designed empirical study.
    \end{itemize}
\end{itemize}

\subsection*{7. Limitations}
\begin{itemize}
    \item \textbf{Already improved:}
    \begin{itemize}
        \item The manuscript already acknowledges the main limitations clearly:
        \begin{itemize}
            \item tiny paired cohort,
            \item tiny validation split,
            \item CT imbalance,
            \item unstable MHRA behavior,
            \item and lack of strong benchmarking validity.
        \end{itemize}
        \item The appendix further reinforces these limitations with concrete patient counts, per-seed instability, and confusion-pattern analysis.
    \end{itemize}
    \item \textbf{Still needs revision:}
    \begin{itemize}
        \item The limitations are currently embedded mostly inside discussion-style prose.
        \item For top-tier style, they should be turned into a more \textbf{structured limitations-to-future-work bridge}.
        \item Reviewers should leave the paper with the impression that the next steps are concrete and technically justified.
    \end{itemize}
    \item \textbf{Recommended action:}
    \begin{itemize}
        \item Add a dedicated paragraph or subsection that separates limitations into:
        \begin{enumerate}
            \item \textbf{data limitations},
            \item \textbf{method limitations},
            \item \textbf{evaluation limitations}.
        \end{enumerate}
        \item Then map each limitation to a direct future action:
        \begin{itemize}
            \item larger paired cohort,
            \item stronger teacher/student backbones,
            \item simpler and stronger KD baselines,
            \item external evaluation,
            \item and better-calibrated teacher-side feature transfer.
        \end{itemize}
    \end{itemize}
\end{itemize}

\subsection*{Priority Ranking of Remaining Revisions}
\begin{enumerate}
    \item \textbf{Highest priority:} strengthen the paper's contribution framing as a reproducible and cautionary cross-modality pilot study rather than as a performance-driven new SOTA method.
    \item \textbf{High priority:} sharpen the novelty/gap statement in the Introduction and Related Work.
    \item \textbf{High priority:} add stronger design rationale in the Method section, especially for MHRA.
    \item \textbf{High priority:} reorganize the Results section around transferable empirical lessons.
    \item \textbf{Medium priority:} make the Limitations section more structured and forward-linked.
    \item \textbf{Medium priority:} polish wording to further reduce any residual ambiguity between ``proposed validated module'' and ``currently tested hypothesis''.
\end{enumerate}

\subsection*{Bottom-Line Recommendation}
\begin{itemize}
    \item The revised manuscript is \textbf{substantially better than before} and is now much closer to a defensible submission.
    \item However, under strict top-tier standards, it is \textbf{not yet optimal} mainly because:
    \begin{itemize}
        \item the novelty positioning is still not maximally sharp,
        \item the method rationale is still somewhat under-argued,
        \item and the experimental evidence remains too small to support a strong method-driven claim.
    \end{itemize}
    \item Therefore, the next revision should focus less on adding decorative text and more on:
    \begin{quote}
        sharpening contribution logic, clarifying methodological necessity, and converting the current negative/neutral findings into a strong reproducible scientific message.
    \end{quote}
\end{itemize}
